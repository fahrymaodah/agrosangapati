# GitHub Copilot Instructions for Filament 4.x Project

## Project Context
This is a Laravel 11 application using Filament 4.1.10 for admin panels. Follow these guidelines strictly:

## Filament 4.x Namespace Rules
1. **Layout components** (Section, Grid, Fieldset) → `Filament\Schemas\Components\*`
2. **Form inputs** (TextInput, Select, etc.) → `Filament\Forms\Components\*`  
3. **Infolist entries** (TextEntry, ImageEntry, etc.) → `Filament\Infolists\Components\*`
4. **Form helpers** (Get) → `Filament\Forms\Get`
5. **Base schema** → `Filament\Schemas\Schema`

## Forbidden Usage
- ❌ `Filament\Forms\Components\Section` 
- ❌ `Filament\Infolists\Components\Section`
- ❌ `Filament\Infolists\Components\Grid`
- ❌ `Filament\Infolists\Components\Split` (doesn't exist)
- ❌ `TextEntry::description()` (use `helperText()`)

## Project Structure
- Resources: `app/Filament/Resources/{Entity}/{EntityResource}.php`
- Schemas: `app/Filament/Resources/{Entity}/Schemas/{Type}.php`
- Models: `app/Models/{Model}.php`

## Code Style
- Use strict types: `declare(strict_types=1);`
- Follow Laravel conventions
- Use proper relationship eager loading
- Always clear cache after schema changes
